# Domain Optimization Skill

对既有领域经验做迭代优化，使其更贴近真实访谈沉淀，同时保持可复用性。

输入来自 payload，常见字段：
- original_domain_experience：原始经验文本；
- project_structure：项目最终结构（含主题与槽位值）。

优化方法：
1. 先抽取原经验骨架：核心目标、关键维度、重要约束、注意事项；
2. 从 project_structure 提取真实实践证据：高频槽值、常见追问点、冲突与遗漏点；
3. 做“保留 + 替换 + 增补”：
   - 保留：仍有效且通用的核心框架；
   - 替换：空泛、过时、与实践冲突的表述；
   - 增补：实践中反复出现但原经验缺失的关键细节；
4. 统一术语与量纲，让表达可直接复用于下一批项目；
5. 清理项目专有指代，保留领域可迁移知识。

决策优先级：
- 明确实践证据 > 抽象经验判断；
- 可执行细节 > 空泛原则；
- 稳定共性 > 偶发个例；
- 一致术语 > 多套同义表达。

质量要求：
- 不能偏离原经验主题边界；
- 不能仅做同义改写，必须体现“被实践修正后的信息增量”；
- 不能引入无依据结论；
- 优化后的文本应更利于后续框架生成与访谈提问。
