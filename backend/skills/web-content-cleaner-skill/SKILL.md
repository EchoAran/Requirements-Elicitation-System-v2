# Web Content Cleaner Skill

对抓取到的网页文本做可用性判断与内容清洗，输出可用于需求访谈的有效信息。

输入来自 payload，常见字段：
- project_name；
- initial_requirements；
- url；
- page_text。

执行步骤：
1. 先做可用性筛选：判断页面是否可读、是否与项目需求相关；
2. 若不可用，直接拒绝，不进入正文提炼；
3. 若可用，清除噪声：导航、广告、版权声明、营销话术、无关段落；
4. 从正文中提取与项目需求强相关的信息：业务规则、流程约束、术语定义、风险点、最佳实践；
5. 形成精炼内容，便于后续知识融合和框架生成。

判废标准（任一命中即不可用）：
- 大面积乱码或结构损坏；
- 404、拦截页、登录墙导致无有效正文；
- 纯广告页、目录页、聚合页且缺乏实质信息；
- 与当前项目需求主题基本无关。

提炼原则：
- 只保留“可支持访谈设计”的信息；
- 不引入网页之外的知识；
- 不机械照搬整段原文，去冗余后保留关键语义；
- 术语尽量贴近原文且前后一致。
