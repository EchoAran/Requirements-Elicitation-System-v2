# Interview Info Eval Skill

对输入的项目描述做“信息完备度与可访谈性”评估，用于判断是否需要补充信息。

输入来自 payload，核心字段是 text（待评估文本）。

评估维度：
- goal：目标是否明确，是否包含核心问题与预期价值；
- users：关键对象是否明确，是否说明对象角色与关注点；
- functions：核心议题是否清晰，是否可拆为可访谈主题；
- constraints：边界条件是否具体（时间/预算/合规/资源等）；
- acceptance：成功标准是否可验证、可度量。

评分思路：
1. 先抽取文本中的事实信息，再映射到五个维度；
2. 对每个维度判断覆盖程度：完整覆盖 / 部分覆盖 / 未覆盖；
3. 覆盖越完整、表述越具体、可执行性越高，得分越高；
4. 出现“口号式目标、泛化需求、不可验证表述”时应降分；
5. 评分时保持一致口径，避免同文本在不同维度重复计分。

summary 生成原则：
- 保留用户原始信息，不擅自补全缺失事实；
- 把分散描述整理为结构化要点，便于后续框架生成；
- 同类信息多项并存时完整保留，不随意合并；
- 模糊信息可以保留，但要与明确信息区分。

reason 与 coverage 要求：
- reason 解释为何得出当前总体评估；
- coverage 说明五个维度哪些已足够、哪些需要补充；
- 表述应直接服务后续“补问什么、先问什么”的决策。
