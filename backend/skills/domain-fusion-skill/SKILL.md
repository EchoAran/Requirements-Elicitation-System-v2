# Domain Fusion Skill

将多条领域经验融合为一段可直接用于“访谈框架设计”的统一经验文本。

输入数据来自 payload，核心是 items 列表。每个 item 通常包含：
- domain_name：领域名称；
- weight：权重（0~1）；
- content：经验正文；
- 可能还有 domain_id 等元字段。

融合方法：
1. 先按 weight 从高到低排序，建立主次信息层级；
2. 提取各条内容中的共同主干：目标、用户、关键议题、约束、风险、验收关注点；
3. 用高权重条目确定主叙述框架，再把中低权重中的“非重复、非冲突、可落地细节”并入；
4. 对重复内容做语义去重，保留信息密度更高、约束更明确、可执行性更强的表达；
5. 对冲突内容按权重决策：
   - 差距明显：采用高权重方案；
   - 差距接近：保留场景化分歧并说明适用前提；
6. 最终文本按“从整体到细节”的顺序组织，确保能被后续框架生成直接吸收。

质量标准：
- 不堆砌来源，不逐条拼接，不出现机械罗列；
- 不引入原始内容中不存在的新事实；
- 保留关键术语的一致性，避免同义概念多种叫法并存；
- 重点服务需求获取与访谈设计，弱化实现细节和宣传性内容。
