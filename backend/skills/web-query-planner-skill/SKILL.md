# Web Query Planner Skill

根据项目信息规划高质量检索词，用于后续联网获取领域知识。

输入来自 payload，常见字段：
- project_name；
- initial_requirements。

规划策略：
1. 从需求中先抽取核心实体：行业、业务对象、关键流程、约束条件、验收诉求；
2. 组合检索意图，覆盖“定义类、方案类、规范类、风险类、实践类”信息；
3. 兼顾广度与精度，避免所有查询都落在同一维度；
4. 优先可检索、可命中专业资料的表达，不用口语化长句。

质量要求：
- 查询词应直接服务当前项目，不泛化到无关领域；
- 关键词要包含足够上下文，减少歧义命中；
- 适当加入行业限定词、场景限定词、约束限定词；
- 避免重复语义或仅改写同一句话。

失败规避：
- 不生成空泛词（如“项目管理怎么做”）；
- 不生成与需求矛盾的检索方向；
- 不生成过长、难命中的自然语言段落。
