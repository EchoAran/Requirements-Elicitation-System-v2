# Topic Dependency Skill

识别主题之间的强前置依赖关系，用于后续优先级排序。

输入来自 payload，常见字段：
- topics：主题列表（含 topic_number/topic_content）。

依赖定义：
- source：前置主题。没有它，target 难以高质量推进；
- target：后置主题。其关键判断依赖 source 已采集的信息。

判断方法：
1. 先按语义将主题归类为基础层、业务层、约束层、验收层等；
2. 判断“信息必要性”而非“讨论先后习惯”；
3. 只保留强依赖边：缺少 source 会显著影响 target 的可问性或可判定性；
4. 对弱相关或可并行主题，不强行建立依赖。

约束规则：
- source/target 只能使用给定 topic_number；
- 禁止自依赖、重复依赖、环形依赖；
- 不得凭空新增主题编号；
- 当两主题互有引用但可并行采集时，不建立双向边。

质量标准：
- 依赖边数量应“足够解释顺序”而非“越多越好”；
- 每条边都应可被一句业务逻辑解释；
- 结果应有助于将基础主题排前、衍生主题排后。
