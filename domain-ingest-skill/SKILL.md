# Domain Ingest Skill

将项目材料（文档文本、说明、背景）提炼为可复用的领域经验。

输入来自 payload，常见字段包括：
- domain_name：目标领域名称；
- domain_description：领域简述；
- documents：文档文本数组，每项是一段原始材料。

处理流程：
1. 先通读 documents，筛出与“需求获取/需求澄清/访谈设计”直接相关的内容；
2. 对材料做主题聚类：业务目标、用户与角色、关键流程、核心约束、风险点、验收判据；
3. 合并重复信息，保留跨文档一致且可落地的结论；
4. 对冲突描述按“具体优先、近期优先、可执行优先”选择，必要时保留条件化表述；
5. 形成一段完整经验正文，要求可直接指导后续访谈框架生成。

提炼原则：
- 忠于材料，不臆造事实，不添加文档中不存在的指标；
- 强调可访谈、可追问、可结构化沉淀的信息；
- 弱化宣传语、背景噪音、实现细枝末节；
- 术语统一，避免同义词混用导致歧义；
- 内容应同时包含“共性方法”与“该领域特有注意点”。

标签建议：
- 从正文中抽取 3~8 个高信息量标签；
- 标签要可检索、可复用，避免“通用词+空泛词”。
