# Domain Knowledge Generator Skill

基于项目名称与项目需求，生成一条“可用于访谈框架设计”的领域知识卡片。

输入来自 payload，常见字段：
- project_name 项目名称；
- initial_requirements 项目初始需求；

生成目标：
- 给出该项目场景下最有价值的一条领域经验；
- 这条经验应能帮助后续快速确定访谈章节、主题与关键信息槽；
- 内容要同时覆盖业务逻辑、风险点、约束边界与可落地建议。

内容组织建议：
1. 先识别项目类型与业务主线；
2. 抽取最关键的用户角色与目标冲突；
3. 明确 3~6 个访谈重点维度（如流程、规则、数据、权限、合规、验收）；
4. 给出每个维度下“应重点追问的信息”；
5. 用简洁术语总结为可复用经验。

质量要求：
- 强相关：只保留与当前项目需求密切相关的信息；
- 高密度：避免套话与泛化建议；
- 可迁移：不绑定单一项目私有细节，但保留行业特征；
- 一致性：术语统一，避免前后概念漂移；
- 可执行：读完后能直接指导访谈提问设计。
