# Knowledge Fusion Skill

将多来源知识项融合为一段高可用的领域背景文本，用于后续访谈框架生成。

输入来自 payload，常见字段：
- project_name 项目名称；
- initial_requirements 项目初始需求；
- knowledge_items 知识项列表，（来源可能包括库内检索、联网检索、模型生成、文件解析）。

融合步骤：
1. 先按相关度与可信度排序，优先处理与项目需求强相关的条目；
2. 将条目映射到统一结构：业务目标、用户场景、关键流程、约束风险、验收关注点；
3. 去重并合并相同结论，保留信息增量；
4. 对冲突内容做条件化整合，保留前提差异；
5. 输出一段连贯文本，能够直接指导“章节/主题/槽位”设计。

处理原则：
- 不做来源罗列式拼接，不生成“摘要列表感”文本；
- 不引入无依据新事实，不放大低可信零散信息；
- 保留对访谈设计最有用的信息：可提问、可验证、可落地；
- 弱化营销口径、术语噪音和实现细枝末节；
- 术语统一，避免同一概念多套表述。
