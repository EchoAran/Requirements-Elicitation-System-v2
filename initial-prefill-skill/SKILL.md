# Initial Prefill Skill

基于项目初始描述，对既有槽位进行首轮预填充，目标是减少后续重复提问。

输入来自 payload，常见字段：
- initial_requirements：项目初始文本；
- topics_list：主题清单；
- slots_by_topic：按主题组织的槽位清单。

执行要求：
1. 只在 initial_requirements 中有明确依据时填值；
2. 不得新增、删除或改名任何槽位；
3. 无法确定时保持空缺，不做猜测性补全；
4. 一个槽位可容纳多个并列信息时，要完整保留，不漏项。

提取原则：
- 字面优先：以原文可证据化的信息为准；
- 准确优先：宁可少填，也不填模糊推断；
- 原意优先：尽量保持用户表述的语义，不做改写扩写；
- 相关优先：仅填与该槽位语义高度匹配的信息。

冲突处理：
- 若同一槽位在文本中出现多个版本，优先更具体、更新近、约束更强的描述；
- 若无法判断优先级，保留更保守表述，避免引入错误细节。

一致性检查：
- topic_number/slot_number 必须与输入中已有编号一致；
- 不把某主题的信息误填到其他主题槽位；
- 不回填与访谈目标弱相关的噪声信息。
